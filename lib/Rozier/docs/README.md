# Rozier docs

This folder is a shared reference for developers and LLM agents working on the
Rozier backoffice. It explains the scope of the frontend codebase, how it fits
into the monorepo, and the team conventions for implementing changes.

Use it as a quick entry point when you touch anything in `lib/Rozier`.

## What this folder covers

- Purpose and boundaries of the Rozier UI within the monorepo
- Codebase navigation and build pipeline notes
- Team engineering practices and workflow
- Current roadmap and modernization goals
- LLM-specific guidance and safe defaults

## Start here

- `context.md`: what Rozier is, how it is built, and where outputs land
- `engineering-practices.md`: how we work, review, test, and ship
- `roadmap.md`: current modernization work and priorities
- `llm-guidelines.md`: guidance for automated code changes

## What this folder is not

- End-user documentation or release notes
- Detailed UI specs or design assets
- Backend or bundle-level architecture reference
